# AGENTS.md

## Repo layout

- **Single-file app** — everything lives in `index.html` (HTML + CSS + JS inline). No external CSS/JS files, no build step.
- `data/products.json` is a **unused template** — ignore it. Real product data is a JS array inside `index.html` (~line 2132 on main).
- **`dev`** = working branch. **`main`** = old vibe-coded version (keep for reference).

## Adding products

Edit the `products` array in `index.html`. Each entry needs: `id`, `name`, `type`, `category`, `price`, `badge` (null/new/sale/popular), `rating`, `reviews`, `description`, `colors[]` (with `name` + `hex`), `sizes[]`, `dimensions`, `visualType`, `visualColor`.

Valid `visualType` values: `sofa`, `table`, `bookcase`, `bed`, `chair`, `outdoor_table`, `lamp`, `shelving`, `cushion`, `armchair`, `dining_table`, `table_lamp`. These map to SVGs generated by `getProductSVG()`.

Valid categories: `seating`, `tables`, `storage`, `bedroom`, `lighting`, `decor`, `outdoor`.

## UI features to preserve

- Category nav + filter chips + sort dropdown
- Product grid with hover effects, badges, color dots, wishlist button
- Product detail modal with color/size variants, quantity selector, dimensions
- Cart sidebar (local state only — no checkout/payment)
- Search bar (filters by name, type, category, description)
- Toast notifications
- Fully responsive (1024px / 768px / 480px breakpoints)

## Gotchas

- **No external assets** — no images, no font imports, no CDN dependencies. Everything is self-contained.
- **No persistence** — cart/wishlist reset on page reload. That's intentional (browse-only site).
- **Checkout button** is a placeholder toast — do not implement real payment.
- If editing `index.html`, the CSS is in the `<style>` block and JS is in the `<script>` block. Don't split them into external files unless explicitly asked.
